Retrieval Queries in SQL
’ü«   SQL has one basic statement for retrieving information
    from a database; the SELECT statement
    ’ü«   This is not the same as the SELECT operation of the
        relational algebra
’ü«   Important distinction between SQL and the formal
    relational model:
    ’ü«   SQL allows a table (relation) to have two or more tuples that
        are identical in all their attribute values
    ’ü«   Hence, an SQL relation (table) is a multi-set (sometimes
        called a bag) of tuples; it is not a set of tuples
’ü«   SQL relations can be constrained to be sets by specifying
    PRIMARY KEY or UNIQUE attributes, or by using the
    DISTINCT option in a query

                                                               Slide 2- 18
Retrieval Queries in SQL (contd.)
’ü«   A bag or multi-set is like a set, but an element
    may appear more than once.
    ’ü«   Example: {A, B, C, A} is a bag. {A, B, C} is also a
        bag that also is a set.
    ’ü«   Bags also resemble lists, but the order is irrelevant
        in a bag.
’ü«   Example:
    ’ü«   {A, B, A} = {B, A, A} as bags
    ’ü«   However, [A, B, A] is not equal to [B, A, A] as lists


                                                         Slide 2- 19
Retrieval Queries in SQL (contd.)
’ü«   Basic form of the SQL SELECT statement is called a
    mapping or a SELECT-FROM-WHERE block

SELECT           <attribute list>
  FROM           <table list>
  WHERE          <condition>

    ’ü«   <attribute list> is a list of attribute names whose values are
        to be retrieved by the query
    ’ü«   <table list> is a list of the relation names required to process
        the query
    ’ü«   <condition> is a conditional (Boolean) expression that
        identifies the tuples to be retrieved by the query

ALIASES
’ü«   Some queries need to refer to the same relation twice
     ’ü« In this case, aliases are given to the relation name

’ü«   Query 8: For each employee, retrieve the employee's name, and the
    name of his or her immediate supervisor.

    Q8: SELECT           E.FNAME, E.LNAME, S.FNAME, S.LNAME

UNSPECIFIED
WHERE-clause
’ü«   A missing WHERE-clause indicates no condition; hence,
    all tuples of the relations in the FROM-clause are selected
    ’ü«   This is equivalent to the condition WHERE TRUE
’ü«   Query 9: Retrieve the SSN values for all employees.

    ’ü«   Q9:     SELECT        SSN
                FROM          EMPLOYEE

’ü«   If more than one relation is specified in the FROM-clause
    and there is no join condition, then the CARTESIAN
    PRODUCT of tuples is selected

SET OPERATIONS
’ü«   SQL has directly incorporated some set operations
’ü«   There is a union operation (UNION), and in some versions
    of SQL there are set difference (MINUS) and intersection
    (INTERSECT) operations
’ü«   The resulting relations of these set operations are sets of
    tuples; duplicate tuples are eliminated from the result
’ü«   The set operations apply only to union compatible
    relations; the two relations must have the same attributes
    and the attributes must appear in the same order

NESTING OF QUERIES
’ü«   A complete SELECT query, called a nested query, can be
    specified within the WHERE-clause of another query,
    called the outer query
    ’ü«   Many of the previous queries can be specified in an
        alternative form using nesting
’ü«   Query 1: Retrieve the name and address of all employees
    who work for the 'Research' department.

    Q1:SELECT           FNAME, LNAME, ADDRESS
       FROM             EMPLOYEE
       WHERE            DNO IN (SELECT DNUMBER
       FROM             DEPARTMENT
       WHERE            DNAME='Research' )



                                                              Slide 2- 36
NESTING OF QUERIES (contd.)
’ü«   The nested query selects the number of the 'Research'
    department
’ü«   The outer query select an EMPLOYEE tuple if its DNO
    value is in the result of either nested query
’ü«   The comparison operator IN compares a value v with a
    set (or multi-set) of values V, and evaluates to TRUE if v
    is one of the elements in V
’ü«   In general, we can have several levels of nested queries
’ü«   A reference to an unqualified attribute refers to the
    relation declared in the innermost nested query
’ü«   In this example, the nested query is not correlated with
    the outer query

                                                          Slide 2- 37
CORRELATED NESTED QUERIES
’ü«   If a condition in the WHERE-clause of a nested query references an
    attribute of a relation declared in the outer query, the two queries are
    said to be correlated
      ’ü« The result of a correlated nested query is different for each tuple

         (or combination of tuples) of the relation(s) the outer query
’ü«   Query 12: Retrieve the name of each employee who has a dependent
    with the same first name as the employee.

    Q12: SELECT           E.FNAME, E.LNAME
        FROM              EMPLOYEE AS E
        WHERE             E.SSN IN
                                 (SELECT    ESSN
                                 FROM       DEPENDENT
                                 WHERE      ESSN=E.SSN AND
                                 E.FNAME=DEPENDENT_NAME)


                                                                     Slide 2- 38
CORRELATED NESTED QUERIES
(contd.)
’ü«   In Q12, the nested query has a different result in the outer
    query
’ü«   A query written with nested SELECT... FROM...
    WHERE... blocks and using the = or IN comparison
    operators can always be expressed as a single block
    query. For example, Q12 may be written as in Q12A

       Q12A:   SELECT        E.FNAME, E.LNAME
               FROM          EMPLOYEE E, DEPENDENT D
               WHERE         E.SSN=D.ESSN AND
                             E.FNAME=D.DEPENDENT_NAME

NULLS IN SQL QUERIES
’ü«   SQL allows queries that check if a value is NULL (missing
    or undefined or not applicable)
’ü«   SQL uses IS or IS NOT to compare NULLs because it
    considers each NULL value distinct from other NULL
    values, so equality comparison is not appropriate.
’ü«   Query 14: Retrieve the names of all employees who do
    not have supervisors.
    Q14:          SELECT          FNAME, LNAME
                  FROM            EMPLOYEE
                  WHERE           SUPERSSN IS NULL
    ’ü«   Note: If a join condition is specified, tuples with NULL values
        for the join attributes are not included in the result

GROUPING
’ü«   In many cases, we want to apply the aggregate
    functions to subgroups of tuples in a relation
’ü«   Each subgroup of tuples consists of the set of
    tuples that have the same value for the grouping
    attribute(s)
’ü«   The function is applied to each subgroup

’ü«   GROUP BY specifies grouping attributes
’ü«   HAVING specifies a condition for selection of groups
’ü«   ORDER BY specifies an order for displaying the result of a query
     ’ü« A query is evaluated by first applying the WHERE-clause, then
