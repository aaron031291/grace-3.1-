### Genesis Comprehensive Tracking System

**Complete Audit Trail for ALL Inputs and Actions**

The Genesis system now tracks **everything** that happens in Grace:

## What Gets Tracked

### 1. User Interactions
- **User Input**: Messages, commands, requests
- **User Uploads**: Files uploaded with full metadata

### 2. AI/Agent Actions
- **AI Responses**: All AI-generated responses with model info and token usage
- **AI Code Generation**: Code generated by AI with purpose and context
- **Coding Agent Actions**: Autonomous agent actions (refactors, fixes, tests)

### 3. External Interactions
- **External API Calls**: Calls to external services (GitHub, ArXiv, etc.)
- **Web Fetches**: HTML/web content fetched with full content metadata

### 4. File Operations
- **File Operations**: Create, update, delete operations
- **File Ingestion**: Files ingested into knowledge base with chunk/embedding counts

### 5. Data Operations
- **Database Changes**: All database INSERT, UPDATE, DELETE operations
- **Librarian Actions**: Auto-categorization, tagging, relationship detection

### 6. System Operations
- **Configuration**: Config changes
- **System Events**: System-level events and errors

## Architecture

```
User Action
    ↓
ComprehensiveTracker
    ↓
Genesis Key Created
    ↓
Parent/Child Linking
    ↓
Complete Audit Trail
```

## Key Features

### 1. Parent-Child Linking
Every action can reference its parent, creating complete chains:

```
User Upload → File Ingestion → AI Extraction → Librarian Processing
   ↓              ↓                ↓                    ↓
Genesis Key 1  Genesis Key 2   Genesis Key 3     Genesis Key 4
```

### 2. Session Tracking
All actions in a session are linked:

```python
with SessionTracker("user-session") as tracker:
    tracker.track_user_input(...)
    tracker.track_ai_response(...)
    tracker.track_file_ingestion(...)
    # All linked to same session
```

### 3. Complete Metadata
Every Genesis Key captures:
- **What**: What happened
- **Where**: Location (file path, URL, table name)
- **When**: Timestamp
- **Who**: Actor (user, AI, system)
- **Why**: Purpose/reason
- **How**: Method used
- **Input/Output**: Full input and output data
- **Tags**: Searchable tags for querying

### 4. Timeline Reconstruction
Get complete timeline of any session:

```python
tracker = ComprehensiveTracker(session_id="abc123")
timeline = tracker.get_session_timeline()

for event in timeline:
    print(f"{event.when_timestamp}: {event.what_description}")
```

## Usage Examples

### Track User Upload

```python
from genesis.comprehensive_tracker import ComprehensiveTracker

tracker = ComprehensiveTracker(db_session=db, user_id="user_12345")

# Track upload
upload_key = tracker.track_user_upload(
    filename="document.pdf",
    file_path="/knowledge_base/document.pdf",
    file_size=1024768,
    file_type="application/pdf"
)

# Track ingestion (linked to upload)
ingestion_key = tracker.track_file_ingestion(
    filename="document.pdf",
    file_path="/knowledge_base/document.pdf",
    document_id=123,
    chunks_created=25,
    embeddings_created=25,
    parent_key_id=upload_key.key_id  # Link to upload
)

# Track librarian processing (linked to ingestion)
librarian_key = tracker.track_librarian_action(
    document_id=123,
    action_type="auto_categorize",
    tags_assigned=["pdf", "research"],
    relationships_detected=3,
    rules_matched=["PDF Documents"],
    parent_key_id=ingestion_key.key_id  # Link to ingestion
)

# Result: Complete chain tracked
# Upload → Ingestion → Librarian
```

## Database Schema

### GenesisKey Table
```sql
- key_id: Unique identifier
- parent_key_id: Links to parent action
- key_type: Type of action (USER_INPUT, AI_RESPONSE, etc.)
- status: Status (ACTIVE, ARCHIVED, etc.)
- user_id: User performing action
- session_id: Session identifier
- what_description: What happened
- where_location: Where it happened
- when_timestamp: When it happened
- why_reason: Why it happened
- who_actor: Who performed it
- how_method: How it was done
- input_data: Input data (JSON)
- output_data: Output data (JSON)
- context_data: Additional context (JSON)
- tags: Searchable tags (JSON)
- file_path: File path (if applicable)
- code_before/code_after: Code changes
- error info: Error tracking
- fix info: Fix tracking
```
