## trust_engine.py
"\nTrust Engine — Component-level KPI tracking with trust scoring,\nverification, and autonomous remediation.\n\nHow it works:\n1. Every component/model/mechanism is tracked individually\n2. Each component's output is chunked and scored out of 100\n3. Scores aggregate into component trust (0-100)\n4. Component trust aggregates into system trust\n5. Trust determines verification requirements:\n   - 80+: deterministically verified (internal data) → auto-accept\n   - 60-79: LLM-driven → verify against knowledge base\n   - 40-59: unknown source → verify against web + APIs + oracle\n   - <40: reject or require human approval\n6. Failed components trigger self-healing or coding agent\n7. Genesis keys track every output for provenance\n\nVerification sources (checked in order):\n  1. Internal: knowledge base, memory mesh, chats, projects\n  2. Oracle: training data, learned patterns\n  3. External: web search, APIs, whitelist sources\n"
